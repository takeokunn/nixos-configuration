# ececute - タスク遂行コマンド

以下の点に注意して、詳細な指示のもとサブエージェントに実作業を依頼し、タスクを遂行してください。この内容は基本的に優先される事項です。

- プロジェクトの整理・統括を行うことに注力し、サブエージェントを最大限活用すること
- タスクが大きすぎる場合は、適切な大きさのタスクに分割してタスクを整理してから依頼すること
- タスク全体の依存関係を整理し、並列可能なタスクは、複数のサブエージェントに並列で依頼すること
- 各サブエージェントへの指示は可能な限り詳細に記述すること。そして、serena mcp, context7 mcp を積極利用する旨の指示を含めること
- サブエージェントでのcodex mcp利用について:
  - **タイムアウト回避が最優先**: codex mcpは必須ではなく、タイムアウトリスクがある場合は使用しない
  - **基本ツール優先**: 調査（Read/Grep/Glob）や編集（Edit/Write）など基本ツールで完結できる場合はcodexを使用しない
  - **最小粒度の原則**: codexを使う場合、1回の呼び出しは1つの明確で小さなタスクのみ（例: 単一ファイルの特定関数の修正のみ）
  - **段階的実行**: 複雑な作業は「調査」「設計検討」「実装」の各フェーズを別々のcodex呼び出しに分離
  - **複数ファイル編集の禁止**: codex内で複数ファイルを同時編集しない。ファイルごとに別の呼び出しに分割
- 与えられた指示に集中すること
- 過去の実装を示唆するコメント、不必要なコメントは避けること（ユーザーは真に意味のあるコメントのみを求めている）
